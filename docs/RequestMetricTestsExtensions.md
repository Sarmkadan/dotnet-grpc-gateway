# RequestMetricTestsExtensions

The `RequestMetricTestsExtensions` class provides a set of static helper methods designed to facilitate unit testing within the `dotnet-grpc-gateway` project. It serves as a factory for generating `RequestMetric` instances with specific, predetermined characteristics, allowing developers to quickly construct test data for valid requests, various error conditions, performance anomalies, and retry scenarios without manually instantiating complex objects.

## API

### `CreateValidMetric`
Generates a `RequestMetric` instance representing a successful gRPC request with standard properties.
*   **Parameters**: None.
*   **Return Value**: A `RequestMetric` object with a valid service name, successful status code, and positive duration.
*   **Throws**: None.

### `CreateInvalidServiceNameMetric`
Generates a `RequestMetric` instance where the service name is malformed or missing to test validation logic.
*   **Parameters**: None.
*   **Return Value**: A `RequestMetric` object containing an invalid or empty service name string.
*   **Throws**: None.

### `CreateNegativeDurationMetric`
Generates a `RequestMetric` instance with an impossible negative duration value, useful for testing data sanitization or overflow handling.
*   **Parameters**: None.
*   **Return Value**: A `RequestMetric` object where the duration property is set to a negative `TimeSpan`.
*   **Throws**: None.

### `CreateDurationSequence`
Generates an enumerable sequence of `RequestMetric` objects with incrementing or varying durations, typically used for testing aggregation, averaging, or percentile calculations.
*   **Parameters**: None.
*   **Return Value**: An `IEnumerable<RequestMetric>` containing multiple metric instances with progressive duration values.
*   **Throws**: None.

### `CreateSlowRequest`
Generates a `RequestMetric` instance representing a request that exceeded defined latency thresholds.
*   **Parameters**: None.
*   **Return Value**: A `RequestMetric` object with a duration significantly higher than the standard SLA limit.
*   **Throws**: None.

### `CreateErrorMetric`
Generates a `RequestMetric` instance simulating a failed request with a non-success gRPC status code.
*   **Parameters**: None.
*   **Return Value**: A `RequestMetric` object configured with an error status code (e.g., `Internal`, `Unavailable`) and associated error details.
*   **Throws**: None.

### `CreateRetryMetric`
Generates a `RequestMetric` instance indicating a request that was attempted multiple times.
*   **Parameters**: None.
*   **Return Value**: A `RequestMetric` object with retry count metadata populated to reflect transient failure handling.
*   **Throws**: None.

## Usage

The following example demonstrates how to use the extensions to populate a test list with a mix of successful and erroneous metrics for validation testing:

```csharp
using System.Collections.Generic;
using DotNetGrpcGateway.Metrics;

public class MetricValidationTests
{
    public void Test_Metric_Validation_Logic()
    {
        var metrics = new List<RequestMetric>
        {
            RequestMetricTestsExtensions.CreateValidMetric(),
            RequestMetricTestsExtensions.CreateInvalidServiceNameMetric(),
            RequestMetricTestsExtensions.CreateErrorMetric()
        };

        // Assert validation logic handles the mixed set correctly
        var validCount = metrics.Count(m => m.IsValid);
        Assert.AreEqual(1, validCount);
    }
}
```

This example illustrates generating a sequence of durations to test statistical aggregation functions:

```csharp
using System.Linq;
using DotNetGrpcGateway.Metrics;

public class AggregationTests
{
    public void Test_Average_Duration_Calculation()
    {
        var sequence = RequestMetricTestsExtensions.CreateDurationSequence();
        
        var averageDuration = sequence.Average(m => m.Duration.TotalMilliseconds);
        
        // Verify the average falls within expected bounds based on the sequence logic
        Assert.IsTrue(averageDuration > 0);
        Assert.IsTrue(averageDuration < 5000); 
    }
}
```

## Notes

*   **Immutability**: The methods return new instances of `RequestMetric` on every call. Modifying a returned instance will not affect subsequent calls or other instances generated by these helpers.
*   **Thread Safety**: As the class consists entirely of static methods that do not maintain internal state and rely on creating new objects, all members are inherently thread-safe and can be called concurrently from multiple test threads without synchronization.
*   **Edge Cases**: The `CreateNegativeDurationMetric` method explicitly bypasses logical time constraints to produce invalid data; consumers of this metric must ensure their logic handles negative `TimeSpan` values gracefully to avoid unexpected exceptions during arithmetic operations.
*   **Sequence Consistency**: The `CreateDurationSequence` method generates a deterministic sequence. Relying on specific values within the sequence for assertions is safe, provided the underlying implementation of the sequence generation logic remains unchanged.
