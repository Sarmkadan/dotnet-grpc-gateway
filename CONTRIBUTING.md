# Contributing to dotnet-grpc-gateway

First off, thank you for considering contributing to dotnet-grpc-gateway! It's people like you that make the open source community such a great place to learn, inspire, and create.

## How to Contribute

### 1. Fork and Clone
- Fork the repository on GitHub.
- Clone your fork locally: `git clone https://github.com/your-username/dotnet-grpc-gateway.git`

### 2. Create a Branch
Create a new branch for your feature or bug fix:
`git checkout -b feature/my-awesome-feature`

### 3. Development Requirements
- **.NET 10.0 SDK** or later

### 4. Running Tests
Make sure all tests pass before submitting a Pull Request. Run the following command in the project root:
`dotnet test`

### 5. Submit a Pull Request
Push your branch to your fork and submit a Pull Request against the `main` branch of the original repository.

## Code Style and Guidelines
- Follow the existing code conventions used throughout the project.
- Provide XML documentation comments for public APIs.
- **IMPORTANT:** Keep all existing author headers intact in the source files. DO NOT remove them.

## Reporting Issues
If you find a bug or have a feature request, please use GitHub Issues.
When reporting a bug, please include:
- A descriptive title
- Steps to reproduce the issue
- Expected vs actual behavior
- Relevant logs or error messages
